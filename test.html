⌨️ ورودی و خروجی در پایتون

📤 خروجی (چاپ) با print()

تابع print() برای نمایش اطلاعات در صفحه خروجی (ترمینال یا کنسول) استفاده می‌شود.

🔹 مثال ساده:
print("سلام دنیا")

خروجی:

سلام دنیا
🔹 چاپ چند مقدار با کاما:
name = "Ali"
age = 25
print("نام:", name, "سن:", age)

خروجی:

نام: Ali سن: 25
🔹 استفاده از f-string (فرمت‌گذاری پیشرفته):
name = "Sara"
age = 22
print(f"سلام {name}! سن شما {age} سال است.")

خروجی:

سلام Sara! سن شما 22 سال است.

💡 نکته: f-string از پایتون 3.6 به بعد معرفی شده و بسیار محبوبه.

⚙️ پارامترهای مهم print()

پارامتر کاربرد مثال
sep جداکننده بین موارد print("A", "B", "C", sep="-") → A-B-C
end کاراکتری که بعد از چاپ میاد (پیش‌فرض \n) print("Hello", end=" ")
flush برای اطمینان از چاپ فوری کاربرد در سیستم‌های خاص

📥 ورودی گرفتن با input()

تابع input() از کاربر مقدار دریافت می‌کنه. همیشه مقدار را به صورت رشته (str) برمی‌گردونه.

🔹 مثال ساده:
name = input("نام شما چیست؟ ")
print("سلام", name)
🔹 تبدیل ورودی به عدد:
age = int(input("سن خود را وارد کنید: "))
print("سال آینده شما", age + 1, "ساله خواهید بود.")

⚠️ توجه: اگر تبدیل انجام نشه، مقدار هنوز رشته است و نمی‌شه با عددها محاسبه کرد.

🔹 گرفتن عدد اعشاری:
weight = float(input("وزن خود را وارد کنید (کیلوگرم): "))
print("وزن شما:", weight, "کیلوگرم است.")
🔹 گرفتن چند مقدار با یک خط:
x, y = input("دو عدد با فاصله وارد کن: ").split()
x = int(x)
y = int(y)
print("جمع:", x + y)

💪 تمرین‌های پیشنهادی:

  1. از کاربر نام و سن بپرس و به‌صورت سلام {نام}، شما {سن} سال دارید. چاپ کن.
  2. عددی از کاربر بگیر و بگو که آیا آن عدد زوج است یا فرد.
  3. دو عدد اعشاری بگیر و میانگین آن‌ها را محاسبه و چاپ کن.
  4. یک رشته از کاربر بگیر و آن را برعکس (reverse) چاپ کن.
  5. برنامه‌ای بنویس که نمره دانش‌آموز را بگیرد و بگوید آیا قبول شده (>=10) یا نه.

تابع کاربرد

int(input()) تبدیل ورودی به عدد صحیح
float(input()) تبدیل ورودی به عدد اعشاری
f"..." روش قالب‌بندی حرفه‌ای متن در چاپ

💬 کامنت‌گذاری در پایتون

🔍 کامنت چیست؟

کامنت‌ها توضیحاتی هستند که داخل کد نوشته می‌شوند ولی توسط پایتون اجرا نمی‌شوند.

  • فقط برای برنامه‌نویسان و توسعه‌دهندگان هستند
  • برای توضیح منطق کد، راهنما یا غیرفعال کردن موقتی خطوط استفاده می‌شن
  • در خروجی هیچ اثری از کامنت نیست

📝 انواع کامنت در پایتون

1️⃣ کامنت تک‌خطی (Single-line Comment)
# این یک کامنت ساده است
x = 10  # مقدار اولیه برای x

هر چیزی بعد از # در یک خط کامنت حساب می‌شه.

2️⃣ کامنت چندخطی (Multi-line Comment)

در پایتون چیزی به اسم کامنت چندخطی واقعی نداریم مثل زبان‌های دیگر اما می‌تونیم از سه‌تا کوتیشن استفاده کنیم:

"""
این یک کامنت چندخطی است
که برای توضیح طولانی‌تر استفاده می‌شود.
"""

یا:

'''
این هم کامنت چندخطی با کوتیشن تکی است
'''

توجه: این روش در واقع یک رشته (string) بدون استفاده است. در اجرا مشکلی ایجاد نمی‌کند ولی روش رسمی نیست. برای داکیومنتیشن هم استفاده می‌شود.

🎯 چرا کامنت‌گذاری مهم است؟

دلیل توضیح
فهم راحت‌تر کد مخصوصا در پروژه‌های تیمی یا برای آینده خودت
مستندسازی توضیح ورودی‌ها، خروجی‌ها و عملکرد توابع
دیباگ راحت‌تر می‌تونی خطوطی از کد رو موقتا غیرفعال کنی
حرفه‌ای‌بودن کدی که مستند و خواناست ارزش بیشتری داره

⭐ نکات مهم در کامنت‌گذاری حرفه‌ای:

  1. مختصر، مفید و واضح بنویس
  2. از نوشتن توضیح اضافه برای چیزهای واضح خودداری کن
    • ✅ درست: # تقسیم بر تعداد برای میانگین
    • ❌ نادرست: # تقسیم
  3. بالای توابع، کلاس‌ها و بخش‌های پیچیده توضیح بده
  4. از کامنت برای مشخص‌کردن TODO یا BUG هم استفاده کن

📋 مثال کامل:

# تابعی برای محاسبه فاکتوریل
def factorial(n):
    """
    ورودی: n عدد صحیح
    خروجی: فاکتوریل n
    """
    if n == 0:
        return 1  # فاکتوریل صفر برابر 1 است
    return n * factorial(n - 1)

🏷️ TODO / NOTE / FIXME در کامنت‌ها

برای همکاری بهتر در تیم‌ها از این تگ‌ها استفاده می‌شه:

# TODO: اضافه کردن بررسی برای ورودی منفی
# NOTE: این تابع فقط برای اعداد طبیعی استفاده می‌شه
# FIXME: با ورودی صفر کرش می‌کنه

💪 تمرین پیشنهادی:

  1. یک تابع ساده بنویس که عددی را دو برابر می‌کند. بالا و داخل آن کامنت توضیح بنویس.
  2. تابعی بنویس که لیستی را مرتب می‌کند و کامنت‌گذاری کن که چه الگوریتمی استفاده شده (مثلا Bubble Sort).
  3. برنامه‌ای بنویس که نام و سن را دریافت کند. کامنتی بنویس که هدف برنامه چیست.
  4. یک کلاس ساده مثل Car بنویس و تمام ویژگی‌ها و متدهایش را با کامنت توضیح بده.

📊 جمع‌بندی سریع:

مفهوم توضیح
# برای نوشتن کامنت تک‌خطی استفاده می‌شود
''' ... ''' یا """ ... """ برای نوشتن توضیح چندخطی یا docstring
مزیت کامنت مستندسازی، فهم بهتر، دیباگ راحت‌تر، حرفه‌ای شدن
نکته مهم مختصر، مفید و فقط وقتی لازم است

بانک سوالات - ساختارهای کنترلی

1️⃣ شرطی‌ها (if, elif, else)

  1. عددی از کاربر بگیر، اگر مثبت بود چاپ کن "عدد مثبت است".
  2. بررسی کن آیا عدد وارد شده زوج است یا فرد.
  3. برنامه‌ای بنویس که اگر سن کاربر بالای ۱۸ بود، پیام "ورود مجاز است" چاپ شود.
  4. سه عدد بگیر و بزرگ‌ترین آن‌ها را مشخص کن.
  5. برنامه‌ای بنویس که نمره‌ای بگیرد و براساس نمره، وضعیت (عالی، خوب، قابل قبول، مردود) چاپ کند.
  6. اگر رشته‌ای خالی بود، پیام "هیچ متنی وارد نشده" چاپ کن.
  7. برنامه‌ای بنویس که سال را بگیرد و بررسی کند آیا کبیسه است یا نه.
  8. بررسی کن اگر دو عدد برابر بودند، بنویس "مساوی‌اند"، در غیر این صورت تفاوت را چاپ کن.
  9. برنامه‌ای بنویس که اگر کاربر نام کاربری و رمز درست وارد کرد، وارد شود.
  10. بررسی کن آیا طول یک کلمه بیشتر از ۵ کاراکتر است.
  11. برنامه‌ای بنویس که اگر سن بین ۱۳ تا ۱۹ بود، چاپ کند "نوجوان".
  12. بررسی کن اگر ورودی از نوع عدد صحیح نبود، خطا چاپ شود.
  13. برنامه‌ای که کاربر جنسیت وارد کند، اگر "m" بود بگو "آقا"، اگر "f" بود بگو "خانم".
  14. برنامه‌ای که اگر کاربر چیزی وارد نکرد، برنامه پیام "ورودی خالی" بدهد.
  15. بررسی کن آیا عدد وارد شده مضرب ۳ یا مضرب ۵ هست.

2️⃣ حلقه‌ها (for, while)

  1. با for اعداد ۰ تا ۹ را چاپ کن.
  2. با while از عدد ۱ تا ۵ چاپ کن.
  3. برنامه‌ای بنویس که اعداد بین ۱ تا ۱۰ را دوبرابر کند و چاپ کند.
  4. با for روی لیستی از نام‌ها حلقه بزن و سلام بده.
  5. با while اعداد مثبت را از کاربر بگیر و جمع بزن تا کاربر صفر وارد کند.
  6. برنامه‌ای بنویس که جمع اعداد از ۱ تا ۱۰۰ را محاسبه کند.
  7. برنامه‌ای که کاراکترهای یک رشته را یکی‌یکی چاپ کند.
  8. با for فقط اعداد زوج بین ۱ تا ۲۰ را چاپ کن.
  9. برنامه‌ای بنویس که از کاربر رشته بگیرد و تعداد حروف را بشمارد.
  10. با while بررسی کن کاربر رمز را درست وارد کرده یا نه، تا درست وارد نکرده حلقه ادامه یابد.
  11. لیستی از نمرات را بگیر و تعداد نمرات بالای ۱۵ را بشمار.
  12. با for بررسی کن آیا عدد خاصی در لیست هست یا نه.
  13. برنامه‌ای بنویس که مجموع اعداد فرد بین ۱ تا ۱۰۰ را چاپ کند.
  14. برنامه‌ای که ۵ بار از کاربر عدد بگیرد و میانگین را حساب کند.
  15. با while از کاربر کلمات بگیر، تا وقتی "پایان" نگفته، ادامه دهد.

3️⃣ توقف با break

  1. با for اعداد ۰ تا ۱۰ را چاپ کن ولی وقتی به ۵ رسید، متوقف شو.
  2. برنامه‌ای بنویس که از کاربر عدد بگیرد، وقتی عدد منفی وارد شد، break بزن.
  3. برنامه‌ای که روی یک لیست بگردد و وقتی مقدار خاصی پیدا شد، حلقه متوقف شود.
  4. برنامه‌ای بنویس که عددی از کاربر بگیرد، اگر مضرب ۷ بود، حلقه را متوقف کند.
  5. با while از کاربر کلمات بگیر و اگر کلمه "خروج" وارد شد، break اجرا شود.
  6. برنامه‌ای که بررسی کند آیا عدد خاصی در لیست وجود دارد یا نه، با break.
  7. لیستی از رشته‌ها را بگرد، اگر به "error" رسیدی، حلقه را متوقف کن.
  8. با for روی اعداد بین ۱ تا ۲۰ بگرد و وقتی اولین عدد زوج پیدا شد، break بزن.
  9. حلقه‌ای بساز که از کاربر رمز بگیرد تا زمانی که درست وارد شود یا ۳ بار اشتباه کرده باشد.
  10. برنامه‌ای که کاربر اعداد وارد کند، اگر مجموع آن‌ها از ۱۰۰ بیشتر شد، break.
  11. حلقه‌ای برای چک کردن موجود بودن کالا در لیست، اگر نبود، متوقف شود.
  12. برنامه‌ای که بررسی کند آیا عدد وارد شده تکراری است، اگر بود، break.
  13. از لیست اعداد، اولین عدد منفی را پیدا کن و حلقه را متوقف کن.
  14. برنامه‌ای که در لیست قیمت‌ها اگر قیمتی بالای ۱۰۰۰ بود، حلقه را متوقف کند.
  15. برنامه‌ای که از کاربر جمله بگیرد و اگر کلمه "بس" در آن بود، دیگر سوال نپرسد.

4️⃣ ✅ ادامه با continue

  1. با for اعداد ۰ تا ۵ را چاپ کن، به جز عدد ۲.
  2. برنامه‌ای که فقط نمرات بالای ۱۰ را چاپ کند و بقیه را با continue رد کند.
  3. از کاربر ۵ عدد بگیر، اگر عدد صفر وارد کرد، چاپ نکن (ردش کن).
  4. لیستی از رشته‌ها داشته باش، فقط مواردی که خالی نیستند را چاپ کن.
  5. برنامه‌ای که فقط اعداد فرد بین ۱ تا ۱۰ را چاپ کند.
  6. برنامه‌ای که هنگام خواندن لیست قیمت‌ها، قیمت‌های صفر را رد کند.
  7. با while از کاربر عدد بگیر و اگر عدد منفی وارد کرد، ردش کند و عدد بعدی را بگیرد.
  8. بررسی کن اگر لیستی مقدار None داشت، آن را چاپ نکند.
  9. در لیستی از نام‌ها، فقط نام‌هایی که با حرف "A" شروع می‌شوند را چاپ کن.
  10. از کاربر کلمه بگیر، اگر بیشتر از ۱۰ کاراکتر بود، چاپ نکن (با continue).
  11. برنامه‌ای که حروف صدادار را در رشته رد کند و فقط حروف بی‌صدا را چاپ کند.
  12. برنامه‌ای که هنگام پیمایش لیست اعداد، فقط مقادیر مثبت را نمایش دهد.
  13. برنامه‌ای بنویس که هنگام پردازش لیست اعداد، مقادیر تکراری را با continue رد کند.
  14. برنامه‌ای که از کاربر عدد بگیرد، فقط اعداد زوج را نگه دارد.
  15. حلقه‌ای که ۱۰ بار اجرا شود ولی فقط در تکرارهای فرد کاری انجام دهد.

خلاصه جامع: برنامه‌نویسی شیء‌گرا (OOP)

1️⃣ کلاس (class) چیست؟

کلاس = قالب یا نقشه برای ساخت اشیاء

در کلاس ویژگی‌ها (متغیرها) و رفتارها (متدها/توابع) تعریف می‌شن.

ساختار:

class ClassName:
    def __init__(self, پارامترها):
        self.ویژگی = مقدار

مثال:

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

2️⃣ شیء (object) چیست؟

شیء = نمونه‌ای از یک کلاس

هر شیء داده‌ها و رفتار مخصوص خودش رو داره.

ساخت شیء:

p1 = Person("Ali", 25)

دسترسی به ویژگی‌ها و متدها:

print(p1.name)        # Ali
print(p1.age)         # 25

⚠️ نکات مهم:

مورد توضیح
class تعریف نوع جدید داده (مثل قالب ساخت انسان، ماشین و...)
object نمونه‌ای از کلاس که داده واقعی داره
__init__() تابع سازنده، هنگام ساخت شیء اجرا می‌شه
self اشاره به شیء فعلی در داخل کلاس
چند شیء از یک کلاس بله، می‌تونیم بی‌نهایت شیء بسازیم با داده‌های مختلف

💡 مثال کامل و ساده:

class Car:
    def __init__(self, brand, year):
        self.brand = brand
        self.year = year

car1 = Car("BMW", 2023)
car2 = Car("Toyota", 2020)

print(car1.brand)  # BMW
print(car2.year)   # 2020

بخش: خواندن فایل (Reading Files)

🎯 هدف از خواندن فایل چیست؟

خیلی وقت‌ها داده‌ها به‌صورت فایل متنی (.txt .csv و...) ذخیره شده‌ن و ما باید با استفاده از پایتون اون‌ها رو باز کرده بخونیم و تحلیل کنیم.

📋 مراحل خواندن فایل:

1️⃣ باز کردن فایل با open()

file = open("filename.txt", "r")

"r" یعنی حالت خواندن (read mode).

2️⃣ خواندن محتوا:

  • read(): کل محتوا به صورت یک رشته
  • readline(): فقط یک خط از فایل
  • readlines(): همه خطوط را به صورت لیست از رشته‌ها

3️⃣ بستن فایل با close()

file.close()

همیشه بعد از باز کردن فایل باید اون رو ببندی.

💡 مثال: خواندن کل محتوا با read()

file = open("data.txt", "r")
content = file.read()
print(content)
file.close()

💡 مثال: خواندن خط به خط با readline()

file = open("data.txt", "r")

line1 = file.readline()
line2 = file.readline()

print("خط اول:", line1)
print("خط دوم:", line2)

file.close()

💡 مثال: خواندن همه خطوط با readlines()

file = open("data.txt", "r")
lines = file.readlines()

for line in lines:
    print(line.strip())  # حذف \n از انتهای هر خط

file.close()

🏆 روش حرفه‌ای: استفاده از with (مدیریت خودکار)

with open("data.txt", "r") as file:
    content = file.read()
    print(content)

در این روش:

  • فایل به‌صورت خودکار بسته می‌شه
  • ایمن‌تر و حرفه‌ای‌تره (بهتر برای مصارف کاری)

⚠️ نکات مهم:

مورد توضیح
مسیر فایل اگر فایل کنار فایل پایتون باشه فقط اسم فایل کافیه. وگرنه مسیر کامل بده (C:/folder/file.txt)
حالت خواندن "r" برای فقط خواندن استفاده می‌شه
خط‌های خالی هنگام خواندن هر خط معمولا \n در انتها هست می‌شه با .strip() حذفش کرد
فایل بزرگ برای فایل‌های بزرگ بهتره خط‌به‌خط بخونی تا حافظه پر نشه

📝 تمرین‌های پیشنهادی:

  1. فایلی به نام info.txt بساز و محتوای آن را بخوان و چاپ کن.
  2. فقط خط اول یک فایل را با readline() بخوان.
  3. همه خطوط را با readlines() بخوان و شماره‌گذاری کن.
  4. با استفاده از with محتویات یک فایل را چاپ کن.
  5. فایلی که وجود ندارد را باز کن و خطا را مدیریت کن.

تابع کاربرد

read() خواندن کل محتوا به صورت یک رشته
readline() خواندن یک خط
readlines() خواندن همه خطوط به صورت لیست
close() بستن فایل
with باز کردن امن و اتوماتیک فایل

بخش: نوشتن فایل (Writing Files)

🎯 چرا فایل‌ها رو بنویسیم؟

گاهی نیاز داریم خروجی برنامه رو ذخیره کنیم مثلا:

  • گزارش‌ها (logs)
  • ذخیره اطلاعات کاربر
  • ثبت نتایج پردازش
  • ساخت فایل متنی خروجی (مثل .txt .csv و...)

📋 مراحل نوشتن در فایل:

1️⃣ باز کردن فایل با حالت نوشتن:

file = open("filename.txt", "w")
  • "w" یعنی write نوشتن در فایل
  • اگر فایل وجود نداشته باشه ساخته می‌شه
  • اگر فایل وجود داشته باشه پاک می‌شه و دوباره نوشته می‌شه!

2️⃣ نوشتن با write():

file.write("سلام دنیا\n")
file.write("خط دوم\n")

3️⃣ بستن فایل:

file.close()

🏆 حالت حرفه‌ای: استفاده از with

with open("filename.txt", "w") as file:
    file.write("این فایل با with نوشته شده است.\n")
  • فایل به‌صورت خودکار بسته می‌شه
  • ایمن‌تره و برای استفاده در شرکت‌ها و پروژه‌ها توصیه می‌شه

📝 حالت‌های نوشتن مختلف:

حالت اسم توضیح
"w" write نوشتن از اول فایل (پاک می‌کنه!)
"a" append اضافه کردن به انتهای فایل بدون پاک کردن قبلی
"x" exclusive فقط اگر فایل وجود نداشت ایجاد و بنویس (اگه باشه خطا می‌ده)

💡 مثال: نوشتن ساده با write()

file = open("output.txt", "w")
file.write("این یک خط است.\n")
file.write("این خط دوم است.\n")
file.close()

💡 مثال: با with و حلقه:

lines = ["مقدار اول\n", "مقدار دوم\n", "مقدار سوم\n"]

with open("lines.txt", "w") as f:
    for line in lines:
        f.write(line)

💡 مثال: اضافه کردن به فایل با "a" (append):

with open("output.txt", "a") as f:
    f.write("خط جدیدی به فایل اضافه شد.\n")

💡 نوشتن چند خط با writelines():

lines = ["خط اول\n", "خط دوم\n", "خط سوم\n"]

with open("list.txt", "w") as f:
    f.writelines(lines)

writelines() لیستی از رشته‌ها رو می‌نویسه (بینشون \n نمی‌ذاره خودت باید اضافه کنی)

💡 نوشتن داده‌های عددی یا متغیرها:

name = "علی"
age = 23

with open("info.txt", "w") as f:
    f.write(f"نام: {name}\nسن: {age}\n")

⚠️ نکات مهم:

مورد توضیح
پاک شدن فایل در حالت "w" اگر فایل موجود باشه کاملا پاک می‌شه
استفاده از with همیشه بهتره چون فایل خودکار بسته می‌شه
نوشتن فارسی پایتون پیش‌فرض UTF-8 استفاده می‌کنه مشکلی نیست اما اگه خاص بود: encoding="utf-8"
خط جدید خودت باید \n اضافه کنی write() خودش خط نمی‌شکنه

📝 تمرین‌های پیشنهادی:

  1. فایل hello.txt بساز و داخلش بنویس: سلام دنیا
  2. اطلاعات کاربر (نام و سن) را از ورودی بگیر و در فایل user.txt ذخیره کن
  3. لیستی از نمرات را در فایلی به اسم grades.txt ذخیره کن
  4. با استفاده از append هر بار اجرای برنامه یک خط جدید اضافه کن
  5. سه جمله‌ی دلخواه را در یک فایل بنویس هرکدام در یک خط مجزا

📚 خلاصه سریع:

تابع / حالت کاربرد
open("file.txt", "w") نوشتن از اول فایل (پاک می‌کنه)
open("file.txt", "a") اضافه کردن به انتهای فایل
write("...") نوشتن یک خط رشته
writelines([...]) نوشتن چند خط از لیست
with open(...) as f: باز و بسته‌کردن حرفه‌ای فایل‌ها

with open("log.txt", "a") as f:

🎯 برنامه‌نویسی شیء‌گرا یعنی چی؟

در OOP به جای فقط نوشتن توابع و متغیرها برنامه رو بر اساس اشیاء (Object) می‌سازیم.

🏗️ کلاس چیست؟

کلاس = قالب یا نقشه ساخت اشیاء

  • با تعریف یک کلاس شما یک نوع جدید داده می‌سازی

class ClassName:

class Person:

🎯 شیء (Object) چیست؟

شیء = نمونه (instance) از یک کلاس

با استفاده از یک کلاس ما می‌تونیم اشیاء مختلف با داده‌های مختلف بسازیم.

مثل اینکه کلاس "ماشین" داریم و از اون ماشین‌های مختلف مثل "پراید" "پژو" یا "تویوتا" می‌سازیم.

🔨 نحوه ساخت شیء در پایتون

object_name = ClassName()

اگر کلاس از __init__() استفاده کرده باشه باید مقدارهای لازم رو موقع ساخت شیء وارد کنیم.

💡 مثال: ساخت شیء از کلاس ساده

        self.age = age

# ساخت دو شیء از کلاس Person

p2 = Person("Sara", 25)

print(p1.name)  # خروجی: Ali

در اینجا:

  • p1 و p2 هر کدام یک شیء جدا هستند
  • مقدار name و age برای هر شیء مستقل است

بانک سوالات - عملگرها

1️⃣ ✖️➗➖➕➰ عملگرهای حسابی (% / * - +)

  1. یک عدد از کاربر بگیر و آن را با عدد ۵ جمع کن و چاپ کن.
  2. حاصل ضرب دو عدد ۸ و ۴ را حساب کن.
  3. برنامه‌ای بنویس که عددی از کاربر بگیرد و باقیمانده‌ی تقسیم آن بر ۲ را نمایش دهد.
  4. برنامه‌ای بنویس که مساحت یک مستطیل را با دریافت طول و عرض محاسبه کند.

2️⃣ ⚖️ عملگرهای مقایسه‌ای (==, !=, >, <, >=, <=)

  1. بررسی کن که آیا دو عدد ورودی از کاربر برابرند یا نه.
  2. اگر سن کاربر بیشتر از ۱۸ بود، پیام "مجاز به ورود هستید" را چاپ کن.
  3. برنامه‌ای بنویس که بررسی کند نمره وارد شده کمتر از ۱۰ است یا نه.
  4. دو عدد بگیر و بررسی کن کدام یک بزرگ‌تر است یا آیا مساوی‌اند.

3️⃣ 🧠 عملگرهای منطقی (and, or, not)

  1. بررسی کن که آیا عدد وارد شده بین ۱۰ تا ۲۰ است (با and).
  2. اگر کاربر ایمیل یا شماره موبایل وارد کرده بود، پیام ثبت موفق بده.
  3. بررسی کن اگر کاربر رمز ندارد (not)، پیغام خطا نمایش بده.
  4. اگر نمره بین ۱۲ تا ۲۰ بود و غیبت نداشت، بگو "قبول شدی".

4️⃣ 🎯 عملگرهای عضویتی (in, not in)

  1. بررسی کن آیا حرف "a" در رشته‌ای که از کاربر می‌گیریم وجود دارد یا نه.
  2. لیستی از میوه‌ها بساز و بررسی کن که آیا "apple" در لیست هست.
  3. اگر نام کاربری وارد شده در لیست کاربران نبود، پیام "ثبت‌نام کنید" را چاپ کن.
  4. برنامه‌ای بنویس که اعداد تکراری را حذف کند و فقط اعداد یکتا را نمایش دهد (با set و in).

5️⃣ 🆔 عملگرهای هویتی (is, is not)

  1. بررسی کن که آیا متغیری مقدار None دارد یا نه (با is).
  2. دو لیست با مقدار مشابه بساز، بررسی کن که a == b و a is b چه نتیجه‌ای می‌دهد.
  3. متغیر x = 5 و y = 5 تعریف کن، سپس بررسی کن x is y چه برمی‌گرداند.
  4. برنامه‌ای بنویس که اگر متغیر ورودی is not None بود، با آن عملیات انجام دهد.

🧠 عملگرهای منطقی در پایتون (and, or, not)

عملگرهای منطقی برای ترکیب شرط‌ها یا برگرداندن نتیجه‌های بولی (True/False) استفاده می‌شوند. بسیار پرکاربردند در شرط‌ها (if) و تصمیم‌گیری‌های پیچیده.

✅ ۱. عملگر and (و)

زمانی True است که هر دو شرط درست باشند. اگر یکی از شرط‌ها یا هر دو نادرست باشد، خروجی False می‌شود.

age = 20

if age >= 18 and has_id:
    print("اجازه ورود دارید.")
else:
    print("ورود ممنوع.")

username = ""
email = "test@gmail.com"

if username or email: # رشته غیرخالی True در نظر گرفته می‌شود
    print("حساب معتبر است.")
else:
    print("نام کاربری یا ایمیل وارد نشده است.")

🔁 ترکیب عملگرها:

if (score >= 15 and score <= 20) or passed_last_year:

  2. برنامه‌ای بنویسید که اگر کاربر رمز را فراموش کرده یا ایمیل نداشت، پیام خطا بدهد.
  3. بررسی کنید که اگر کاربر وارد نشده باشد (not)، پیام دعوت به ورود نمایش داده شود.
  4. شرطی بنویسید که اگر سن بین ۱۸ تا ۳۰ بود و مدرک داشت، پیام "واجد شرایط هستید" چاپ شود.
  5. بررسی کنید که اگر کاربر هم شماره موبایل داده باشد و هم ایمیل، فرم ثبت شود. در غیر این صورت هشدار دهد.

🧩 عملگرهای عضویتی در پایتون (in, not in)

این عملگرها بررسی می‌کنند که آیا یک مقدار (عنصر) درون ساختارهای داده‌ای مانند لیست، رشته، دیکشنری و… وجود دارد یا نه.

✅ ۱. عملگر in

برای بررسی اینکه آیا یک مقدار در یک لیست، رشته، دیکشنری یا مجموعه وجود دارد:

🔸 با رشته‌ها:

text = "Python is awesome"

print("Python" in text)    # True
print("java" in text)      # False

🔸 با لیست‌ها:

numbers = [1, 2, 3, 4]

print(3 in numbers)    # True
print(10 in numbers)   # False

🔸 با دیکشنری‌ها:

در دیکشنری، in فقط کلیدها را بررسی می‌کند، نه مقدارها.

info = {"name": "Ali", "age": 25}

print("name" in info)   # True
print("Ali" in info)    # False

✅ ۲. عملگر not in

برعکس in است: بررسی می‌کند که آیا چیزی در لیست/رشته/دیکشنری وجود ندارد.

colors = ["red", "blue", "green"]

print("yellow" not in colors)   # True
print("blue" not in colors)     # False

✅ مثال کاربردی:

برنامه‌ای که لیستی از نام‌های کاربر داشته باشد و بررسی کند که آیا کاربر خاصی در لیست وجود دارد یا نه:

users = ["ali", "sara", "reza"]

username = input("نام کاربری خود را وارد کنید: ")

if username.lower() in users:
    print("خوش آمدید!")
else:
    print("کاربر یافت نشد.")

🧠 عملگرهای هویتی در پایتون (is, is not)

عملگرهای هویتی بررسی می‌کنند که آیا دو متغیر دقیقاً به یک شیء (object) در حافظه اشاره می‌کنند یا نه.

✅ فرق مهم با ==:

  • == → بررسی می‌کند که مقدارها برابرند (محتوا).
  • is → بررسی می‌کند که هویت اشیاء یکی است (آدرس حافظه).

✅ ۱. عملگر is

بررسی می‌کند آیا دو متغیر به یک شیء یکسان اشاره می‌کنند یا نه.

a = [1, 2, 3]
b = a
print(a is b)     # ✅ True

چون b دقیقاً به همان لیست a اشاره می‌کند، نتیجه True است.

✅ ۲. عملگر is not

بررسی می‌کند که آیا دو متغیر به اشیاء متفاوت اشاره می‌کنند یا نه.

x = [1, 2, 3]
y = [1, 2, 3]
print(x is not y)   # ✅ True

گرچه x و y مقدار یکسان دارند، اما هر کدام یک شیء جداگانه‌اند.

🧪 فرق is و == با مثال:

a = [1, 2]
b = [1, 2]

print(a == b)     # True (مقدارها برابرند)
print(a is b)     # False (اشیاء جدا هستند)

✅ استفاده‌های رایج:

🔸 با None (خیلی مهم!):

بررسی می‌کنیم که آیا یک متغیر مقدار ندارد. در این مورد همیشه از is None به‌جای == None استفاده کنید. این بهترین شیوه (best practice) است.

x = None

if x is None:
    print("هیچ مقداری تنظیم نشده است.")

🔸 در بررسی هویت دقیق داده‌ها:

برای عددهای کوچک (بین -5 تا 256)، پایتون ممکن است آن‌ها را کَش کند و از همان آدرس حافظه استفاده کند. ولی برای بقیه این‌طور نیست.

a = 1000
b = 1000
print(a == b)   # True
print(a is b)   # ❌ ممکن است False شود (در بعضی نسخه‌های پایتون)

✅ مثال کاربردی:

برنامه‌ای بنویسید که بررسی کند آیا متغیر ورودی مقدار ندارد:

def check(x):
    if x is None:
        return "مقداری داده نشده"
    else:
        return "مقدار داده شده:", x

print(check(None))      # مقداری داده نشده
print(check(25))        # ('مقدار داده شده:', 25)

📝 تمرین‌ها:

  1. دو متغیر با مقدار لیست یکسان بسازید، بررسی کنید که == و is چه نتایجی می‌دهند.
  2. بررسی کنید که آیا متغیر status مقدار None دارد یا نه (با is).
  3. متغیری تعریف کنید و بررسی کنید که is not None است، سپس ادامه برنامه را اجرا کنید.
  4. سه متغیر بسازید که دوتای آن‌ها به یک لیست اشاره کنند و سومی نه. بررسی کنید کدام‌ها is همدیگر هستند.
  5. برنامه‌ای بنویسید که اگر متغیر ورودی is None بود، مقدار پیش‌فرض ۰ بدهد، وگرنه همان مقدار را برگرداند.

🎯 جمع‌بندی:

عملگرمعنیکاربرد اصلی

۱. متغیر چیست؟

متغیر فضایی در حافظه است که می‌تواند مقداری را در خود نگه دارد. در پایتون نیازی به اعلام نوع متغیر پیش از استفاده نیست:

x = 10         # یک متغیر با نام x و مقدار عدد صحیح ۱۰

  • پایتون پویاست: نوع متغیر در زمان اجرا تعیین می‌شود.
  • نام متغیر باید با حرف یا آندر‌لاین (_) شروع شود و می‌تواند شامل حروف، اعداد و آندرلاین باشد.
  • از کلمات کلیدی پایتون (مثل for، if، def و…) نمی‌توان به‌عنوان نام متغیر استفاده کرد.

۲. نوع داده int (اعداد صحیح)

نمایانگر اعداد بدون قسمت اعشاری است. می‌تواند مثبت، منفی یا صفر باشد و در پایتون محدودیتی برای طول عدد وجود ندارد.

c = 0

۳. نوع داده float (اعداد اعشاری)

۵. نکات مهم و بهترین شیوه‌ها

  • نام‌گذاری خوانا: از نام‌های معنادار و snake_case استفاده کنید (مانند total_price).
  • کامنت‌گذاری: برای توضیح کدهای مهم از # استفاده کنید.
  • خوانایی کد: در اطراف عملگرها فاصله بگذارید: result = a + b.
  • اجتناب از بازتعریف: نام‌های داخلی مثل int یا list را بازنویسی نکنید.

۱. مجموعه (Set) چیست؟

set یک ساختار نامرتب (unordered)، بدون عنصر تکراری و قابل تغییر (mutable) است. با آکولاد {} ساخته می‌شود:

یا با استفاده از تابع set():

۲. ویژگی‌های مهم مجموعه‌ها

ویژگیتوضیح
نامرتبترتیب عناصر مشخص نیست (برخلاف لیست)
بدون تکرارفقط یک نسخه از هر مقدار ذخیره می‌شود
قابل تغییرمی‌توان عضو اضافه/حذف کرد
عناصر باید تغییرناپذیر باشندمثل int, float, str, tuple

مثال:

حذف:

s.discard(10)   # بدون خطا حذف می‌کند اگر وجود نداشته باشد
s.pop()         # حذف تصادفی یک عنصر
s.clear()       # حذف همه عناصر

۴. عملیات ریاضی روی مجموعه‌ها

مروری بر متغیرها و انواع داده‌ها

  • متغیر: یک نام دلخواه که به فضایی در حافظه اشاره می‌کند تا مقداری (عدد، متن، لیست و…) را نگه دارد.
  • اعداد (int, float):
    • int: اعداد صحیح (مثلاً ۵، -۳، ۰)
    • float: اعداد اعشاری (مثلاً ۳.۱۴، -0.5)
  • رشته (str): دنباله‌ای از کاراکترها (حروف، اعداد، نمادها) که داخل کوتیشن نوشته می‌شود، مثل "سلام" یا 'abc123'.
  • بولین (bool): نمایانگر مقدار منطقی، یا True (درست) یا False (نادرست)، برای تصمیم‌گیری‌ها و شرط‌ها.
  • لیست (list): مجموعه‌ای قابل تغییر و مرتب از مقادیر که با [] تعریف می‌شود؛ می‌توان در آن عضو افزود یا حذف کرد.
  • تاپل (tuple): مجموعه‌ای مرتب و تغییرناپذیر از مقادیر که با () تعریف می‌شود؛ پس از ساخت، قابل ویرایش نیست.
  • دیکشنری (dict): ساختار نامرتب از جفت‌های کلید–مقدار, مثلاً { "name": "Ali", "age": 30 }، برای نگهداری داده‌های مرتبط.
  • مجموعه (set): ساختاری نامرتب و بدون تکرار برای ذخیره مقادیر یکتا، مثل {1, 2, 3}؛ مناسب حذف موارد مکرر و عملیات ریاضی مجموعه.

⚙️ مبحث: ساختارهای کنترلی شرطی در پایتون (if, elif, else)

در برنامه‌نویسی، اغلب نیاز داریم برنامه بر اساس یک شرط تصمیم بگیرد. در پایتون این کار با ساختارهای شرطی انجام می‌شود.

✅ ۱. if: اجرای کد وقتی شرط برقرار است

x = 10

if x > 5:
    print("عدد بزرگتر از ۵ است.")

اگر شرط داخل if درست باشد (True)، کد داخل بلوک اجرا می‌شود.

✅ ۲. else: اگر هیچ‌کدام از شرط‌ها درست نبودند

x = 2

if x > 5:
    print("بزرگتر از ۵")
else:
    print("کوچکتر یا مساوی ۵")

else بدون شرط می‌آید و فقط زمانی اجرا می‌شود که هیچ شرط قبلی برقرار نباشد.

✅ ۳. elif: بررسی شرط‌های دیگر (مثل "else if")

score = 15

if score >= 18:
    print("عالی")
elif score >= 12:
    print("قابل قبول")
else:
    print("مردود")

می‌توان هر تعداد elif داشت، ولی فقط اولین شرطی که درست باشد اجرا می‌شود و بقیه نادیده گرفته می‌شوند.

ساختار کلی:

if شرط۱:
    # اگر شرط۱ درست بود
    ...
elif شرط۲:
    # اگر شرط۲ درست بود
    ...
elif شرط۳:
    ...
else:
    # اگر هیچ‌کدام درست نبود
    ...

✅ مثال واقعی:

age = int(input("سن خود را وارد کنید: "))

if age < 13:
    print("کودک هستید.")
elif age < 20:
    print("نوجوان هستید.")
elif age < 60:
    print("بزرگسال هستید.")
else:
    print("سالمند هستید.")

⚠️ نکات مهم:

  1. تورفتگی (indentation) بسیار مهم است: کدی که مربوط به if یا else است، باید با فاصله (space/tab) زیر آن نوشته شود.
  2. همیشه فقط یک مسیر اجرا می‌شود؛ یعنی اگر if اجرا شود، دیگر elif یا else اجرا نمی‌شوند.
  3. می‌توان فقط if داشت، بدون else یا elif.

✅ شرط‌های ترکیبی با and, or, not:

if age > 18 and age < 30:
    print("جوان بالغ")

خلاصه مباحث کنترل جریان برنامه در پایتون

1️⃣ شرطی‌ها: if, elif, else

برای تصمیم‌گیری در برنامه استفاده می‌شود.

کلمه کلیدی کاربرد
if بررسی یک شرط – اگر برقرار باشد، کد اجرا می‌شود.
elif اگر شرط if برقرار نبود، شرط بعدی را بررسی می‌کند.
else اگر هیچ شرطی برقرار نبود، این بخش اجرا می‌شود.
✅ مثال:
score = 17

if score >= 18:
    print("عالی")
elif score >= 12:
    print("قبول")
else:
    print("مردود")

2️⃣ حلقه‌ها: for, while

برای تکرار کردن کدها استفاده می‌شوند.

نوع حلقه کاربرد
for وقتی تعداد دفعات تکرار مشخص است (مثلاً در لیست یا range)
while وقتی می‌خواهیم تا زمانی که یک شرط برقرار است، تکرار کنیم
✅ مثال for:
for i in range(5):
    print(i)
✅ مثال while:
i = 0
while i < 5:
    print(i)
    i += 1

3️⃣ توقف با break

برای خروج فوری از حلقه وقتی شرایط خاصی اتفاق افتاد.

  • فقط یک حلقه را قطع می‌کند.
  • معمولاً با شرط همراه است.
✅ مثال:
for i in range(10):
    if i == 5:
        break
    print(i)

4️⃣ ✅ ادامه با continue

برای رد کردن تکرار فعلی حلقه و رفتن به تکرار بعدی.

  • کدهای بعد از continue اجرا نمی‌شوند.
  • حلقه متوقف نمی‌شود، فقط از آن مرحله عبور می‌کند.
✅ مثال:
for i in range(5):
    if i == 2:
        continue
    print(i)

  1. عددی از کاربر بگیر، اگر مثبت بود چاپ کن "عدد مثبت است".
  2. بررسی کن آیا عدد وارد شده زوج است یا فرد.
  3. برنامه‌ای بنویس که اگر سن کاربر بالای ۱۸ بود، پیام "ورود مجاز است" چاپ شود.
  4. سه عدد بگیر و بزرگ‌ترین آن‌ها را مشخص کن.
  5. برنامه‌ای بنویس که نمره‌ای بگیرد و براساس نمره، وضعیت (عالی، خوب، قابل قبول، مردود) چاپ کند.
  6. اگر رشته‌ای خالی بود، پیام "هیچ متنی وارد نشده" چاپ کن.
  7. برنامه‌ای بنویس که سال را بگیرد و بررسی کند آیا کبیسه است یا نه.
  8. بررسی کن اگر دو عدد برابر بودند، بنویس "مساوی‌اند"، در غیر این صورت تفاوت را چاپ کن.
  9. برنامه‌ای بنویس که اگر کاربر نام کاربری و رمز درست وارد کرد، وارد شود.
  10. بررسی کن آیا طول یک کلمه بیشتر از ۵ کاراکتر است.
  11. برنامه‌ای بنویس که اگر سن بین ۱۳ تا ۱۹ بود، چاپ کند "نوجوان".
  12. بررسی کن اگر ورودی از نوع عدد صحیح نبود، خطا چاپ شود.
  13. برنامه‌ای که کاربر جنسیت وارد کند، اگر "m" بود بگو "آقا"، اگر "f" بود بگو "خانم".
  14. برنامه‌ای که اگر کاربر چیزی وارد نکرد، برنامه پیام "ورودی خالی" بدهد.
  15. بررسی کن آیا عدد وارد شده مضرب ۳ یا مضرب ۵ هست.

2️⃣ حلقه‌ها (for, while)

  1. با for اعداد ۰ تا ۹ را چاپ کن.
  2. با while از عدد ۱ تا ۵ چاپ کن.
  3. برنامه‌ای بنویس که اعداد بین ۱ تا ۱۰ را دوبرابر کند و چاپ کند.
  4. با for روی لیستی از نام‌ها حلقه بزن و سلام بده.
  5. با while اعداد مثبت را از کاربر بگیر و جمع بزن تا کاربر صفر وارد کند.
  6. برنامه‌ای بنویس که جمع اعداد از ۱ تا ۱۰۰ را محاسبه کند.
  7. برنامه‌ای که کاراکترهای یک رشته را یکی‌یکی چاپ کند.
  8. با for فقط اعداد زوج بین ۱ تا ۲۰ را چاپ کن.
  9. برنامه‌ای بنویس که از کاربر رشته بگیرد و تعداد حروف را بشمارد.
  10. با while بررسی کن کاربر رمز را درست وارد کرده یا نه، تا درست وارد نکرده حلقه ادامه یابد.
  11. لیستی از نمرات را بگیر و تعداد نمرات بالای ۱۵ را بشمار.
  12. با for بررسی کن آیا عدد خاصی در لیست هست یا نه.
  13. برنامه‌ای بنویس که مجموع اعداد فرد بین ۱ تا ۱۰۰ را چاپ کند.
  14. برنامه‌ای که ۵ بار از کاربر عدد بگیرد و میانگین را حساب کند.
  15. با while از کاربر کلمات بگیر، تا وقتی "پایان" نگفته، ادامه دهد.

3️⃣ توقف با break

  1. با for اعداد ۰ تا ۱۰ را چاپ کن ولی وقتی به ۵ رسید، متوقف شو.
  2. برنامه‌ای بنویس که از کاربر عدد بگیرد، وقتی عدد منفی وارد شد، break بزن.
  3. برنامه‌ای که روی یک لیست بگردد و وقتی مقدار خاصی پیدا شد، حلقه متوقف شود.
  4. برنامه‌ای بنویس که عددی از کاربر بگیرد، اگر مضرب ۷ بود، حلقه را متوقف کند.
  5. با while از کاربر کلمات بگیر و اگر کلمه "خروج" وارد شد، break اجرا شود.
  6. برنامه‌ای که بررسی کند آیا عدد خاصی در لیست وجود دارد یا نه، با break.
  7. لیستی از رشته‌ها را بگرد، اگر به "error" رسیدی، حلقه را متوقف کن.
  8. با for روی اعداد بین ۱ تا ۲۰ بگرد و وقتی اولین عدد زوج پیدا شد، break بزن.
  9. حلقه‌ای بساز که از کاربر رمز بگیرد تا زمانی که درست وارد شود یا ۳ بار اشتباه کرده باشد.
  10. برنامه‌ای که کاربر اعداد وارد کند، اگر مجموع آن‌ها از ۱۰۰ بیشتر شد، break.
  11. حلقه‌ای برای چک کردن موجود بودن کالا در لیست، اگر نبود، متوقف شود.
  12. برنامه‌ای که بررسی کند آیا عدد وارد شده تکراری است، اگر بود، break.
  13. از لیست اعداد، اولین عدد منفی را پیدا کن و حلقه را متوقف کن.
  14. برنامه‌ای که در لیست قیمت‌ها اگر قیمتی بالای ۱۰۰۰ بود، حلقه را متوقف کند.
  15. برنامه‌ای که از کاربر جمله بگیرد و اگر کلمه "بس" در آن بود، دیگر سوال نپرسد.

4️⃣ ✅ ادامه با continue

  1. با for اعداد ۰ تا ۵ را چاپ کن، به جز عدد ۲.
  2. برنامه‌ای که فقط نمرات بالای ۱۰ را چاپ کند و بقیه را با continue رد کند.
  3. از کاربر ۵ عدد بگیر، اگر عدد صفر وارد کرد، چاپ نکن (ردش کن).
  4. لیستی از رشته‌ها داشته باش، فقط مواردی که خالی نیستند را چاپ کن.
  5. برنامه‌ای که فقط اعداد فرد بین ۱ تا ۱۰ را چاپ کند.
  6. برنامه‌ای که هنگام خواندن لیست قیمت‌ها، قیمت‌های صفر را رد کند.
  7. با while از کاربر عدد بگیر و اگر عدد منفی وارد کرد، ردش کند و عدد بعدی را بگیرد.
  8. بررسی کن اگر لیستی مقدار None داشت، آن را چاپ نکند.
  9. در لیستی از نام‌ها، فقط نام‌هایی که با حرف "A" شروع می‌شوند را چاپ کن.
  10. از کاربر کلمه بگیر، اگر بیشتر از ۱۰ کاراکتر بود، چاپ نکن (با continue).
  11. برنامه‌ای که حروف صدادار را در رشته رد کند و فقط حروف بی‌صدا را چاپ کند.
  12. برنامه‌ای که هنگام پیمایش لیست اعداد، فقط مقادیر مثبت را نمایش دهد.
  13. برنامه‌ای بنویس که هنگام پردازش لیست اعداد، مقادیر تکراری را با continue رد کند.
  14. برنامه‌ای که از کاربر عدد بگیرد، فقط اعداد زوج را نگه دارد.
  15. حلقه‌ای که ۱۰ بار اجرا شود ولی فقط در تکرارهای فرد کاری انجام دهد.

مبحث: توابع در پایتون — تعریف با def

✅ تابع چیست؟

تابع (Function) مجموعه‌ای از کدهاست که یک کار مشخص را انجام می‌دهد. با تعریف تابع می‌توانیم آن کار را چند بار استفاده کنیم، بدون اینکه کد را تکرار کنیم.

️ تعریف تابع با def:

def نام_تابع():
    # کدهایی که اجرا می‌شوند
مثال ساده:
def salam():
    print("سلام! خوش آمدی")

و برای اجرای تابع:

salam()
خروجی:
سلام! خوش آمدی

فواید استفاده از تابع:

  • ✅ جلوگیری از تکرار کد
  • ✅ خوانایی و نظم در برنامه
  • ✅ قابلیت استفاده مجدد
  • ✅ تست راحت‌تر کد
  • ✅ سازماندهی در پروژه‌های بزرگ

دریافت ورودی در تابع (پارامتر):

می‌توان اطلاعات را به‌عنوان پارامتر به تابع داد:

def greet(name):
    print("سلام", name)

greet("علی")
خروجی:
سلام علی

بازگرداندن مقدار (return):

تابع می‌تواند یک مقدار به برنامه بازگرداند:

def jam(a, b):
    return a + b

result = jam(3, 5)
print("نتیجه:", result)
خروجی:
نتیجه: 8

وقتی از return استفاده می‌کنی، تابع متوقف می‌شود و مقدار موردنظر را برمی‌گرداند.

نکات کلیدی:

مورد توضیح
def برای تعریف تابع استفاده می‌شود
() داخل پرانتز، پارامترها تعریف می‌شوند
return خروجی تابع را مشخص می‌کند
نام تابع باید با حروف، زیرخط (_) و بدون فاصله باشد

مثال‌های واقعی:

  1. تابعی بنویس که سن را گرفته و سال تولد را برگرداند.
  2. تابعی که لیستی از نمرات را بگیرد و میانگین را حساب کند.
  3. تابعی بنویس که بررسی کند آیا عدد ورودی زوج است یا نه.
  4. تابعی که نام و نام خانوادگی را گرفته و خوش‌آمدگویی چاپ کند.
  5. تابعی بنویس که دو عدد گرفته و حاصل‌ضرب را برگرداند.

ترکیب توابع با شرط و حلقه:

def check_grade(score):
    if score >= 18:
        return "عالی"
    elif score >= 12:
        return "قبول"
    else:
        return "مردود"

for s in [19, 15, 9]:
    print(check_grade(s))

جمع‌بندی:

کلمه کلیدی کاربرد
def تعریف تابع
() مشخص کردن ورودی‌ها
return بازگرداندن مقدار خروجی
تابع() اجرای تابع

مثال ترکیبی کامل:

def is_even(n):
    if n % 2 == 0:
        return True
    else:
        return False

for i in range(1, 6):
    print(i, "زوج است؟", is_even(i))

🎯 مبحث: پارامترها و آرگومان‌ها در توابع پایتون (Parameters & Arguments)

✅ تعریف سریع:

اصطلاح توضیح ساده
پارامتر متغیرهایی که در تعریف تابع استفاده می‌کنی.
آرگومان مقدارهایی که موقع صدا زدن تابع بهش می‌دی.

🧠 مثال ساده:

def greet(name):  # name = پارامتر
    print("سلام", name)

greet("علی")       # "علی" = آرگومان

🔹 خروجی:

سلام علی

✳️ انواع آرگومان‌ها در پایتون:

1️⃣ آرگومان موقعیتی (Positional Arguments)

بر اساس ترتیب تعریف‌شده پارامترها، مقدارها به تابع داده می‌شوند:

def info(name, age):
    print("نام:", name)
    print("سن:", age)

info("سارا", 25)

2️⃣ آرگومان کلیدی (Keyword Arguments)

نام پارامترها را همراه مقدارشان مشخص می‌کنیم (در این حالت ترتیب مهم نیست):

info(age=25, name="سارا")

3️⃣ پارامتر با مقدار پیش‌فرض (Default Parameters)

می‌توانی برای پارامترها مقدار پیش‌فرض تعیین کنی:

def greet(name="مهمان"):
    print("سلام", name)

greet()         # خروجی: سلام مهمان
greet("رضا")    # خروجی: سلام رضا

4️⃣ پارامترهای متغیر (Variable-Length Arguments)

📌 پارامترهای نامحدود موقعیتی: *args

برای وقتی که تعداد نامشخصی از آرگومان‌ها داری:

def total(*numbers):
    print(sum(numbers))

total(1, 2, 3)       # خروجی: 6
total(10, 20, 30, 40)  # خروجی: 100

*args یک تاپل از همه آرگومان‌ها می‌سازد.

📌 پارامترهای نامحدود کلیدی: **kwargs

وقتی می‌خواهی هر تعداد آرگومان کلیدی بپذیری:

def show_info(**person):
    print("نام:", person.get("name"))
    print("سن:", person.get("age"))

show_info(name="ندا", age=22)

**kwargs یک دیکشنری از پارامترهای کلیدی می‌سازد.

🧪 مثال ترکیبی:

def student_info(name, age=18, *skills, **extra):
    print(f"{name} - {age} ساله")
    print("مهارت‌ها:", skills)
    print("اضافی:", extra)

student_info("حسین", 20, "Python", "Django", city="Tehran", grade="A")

🔹 خروجی:

حسین - 20 ساله
مهارت‌ها: ('Python', 'Django')
اضافی: {'city': 'Tehran', 'grade': 'A'}

📌 قوانین ترتیب پارامترها:

در تعریف تابع، ترتیب پارامترها باید این‌گونه باشد:
۱. پارامترهای معمولی (موقعیتی)
۲. پارامترهای پیش‌فرض
۳. *args
۴. **kwargs

❌ این اشتباه است:

def f(a=1, b):  # خطا می‌دهد
    ...

🧾 جمع‌بندی:

نوع پارامتر نشانه ویژگی
موقعیتی فقط نام ترتیب مهم است
کلیدی name=value ترتیب مهم نیست
پیش‌فرض name=value مقدار دارد حتی اگر چیزی داده نشود
نامحدود (*args) * پیش از نام جمع همه آرگومان‌ها در تاپل
نامحدود کلیدی ** پیش از نام جمع آرگومان‌های کلیدی در دیکشنری

✅ تمرین پیشنهادی:

  1. تابعی بنویس که نام و شهر کاربر را بگیرد و چاپ کند.
  2. تابعی با پارامتر پیش‌فرض بنویس که خوش‌آمدگویی کند.
  3. تابعی با *args بساز که اعداد را گرفته و میانگین حساب کند.
  4. تابعی با **kwargs بساز که اطلاعات فردی (قد، سن، رشته، ...) را چاپ کند.

🔁 مبحث: مقادیر بازگشتی در تابع با return

✅ return چیست؟

وقتی تابعی نیاز دارد مقداری را به جایی که فراخوانی شده برگرداند (مثلاً جمع دو عدد، یا وضعیت قبولی)، از return استفاده می‌کنیم.

📌 ساختار پایه:

def نام_تابع():
   return مقدار

✅ مثال ساده:

def jam(a, b):
   return a + b

x = jam(3, 4)
print("نتیجه:", x)

🔹 خروجی:

نتیجه: 7

🔸 تابع jam مقدار جمع را برمی‌گرداند، نه اینکه فقط آن را چاپ کند.

⚠️ تفاوت print و return:

موضوع print return
نمایش روی صفحه بله خیر (فقط مقدار را بازمی‌گرداند)
استفاده در محاسبات بعدی ❌ نه ✅ بله (می‌توان مقدار را ذخیره یا پردازش کرد)

✅ مثال: محاسبه میانگین

def average(a, b, c):
   return (a + b + c) / 3

result = average(15, 17, 20)
print("میانگین:", result)

✅ نکته مهم:

وقتی return اجرا می‌شود، اجرای تابع در همان لحظه متوقف می‌شود.

خطوط بعد از return در تابع دیگر اجرا نمی‌شوند.

مثال:

def test():
   return "سلام"
   print("این خط اجرا نمی‌شود")

print(test())

✅ تابع بدون return:

اگر توابع مقدار return نداشته باشند، به طور پیش‌فرض None برمی‌گردانند:

def hello():
   print("سلام!")

x = hello()
print(x)  # خروجی: None

✅ بازگشت چند مقدار:

تابع می‌تواند چند مقدار با هم بازگرداند (در قالب تاپل):

def info():
   name = "Ali"
   age = 25
   return name, age

n, a = info()
print(n, a)

🔹 خروجی:

Ali 25

🔁 بازگشت شرطی:

def check(num):
   if num > 0:
       return "مثبت"
   else:
       return "صفر یا منفی"

🧠 کاربردهای رایج return:

  • ✅ بازگرداندن نتیجه‌ی محاسبات
  • ✅ استفاده در شرط‌ها و فیلترها
  • ✅ جدا کردن منطق محاسبه از چاپ
  • ✅ کمک به ساخت توابع تست‌پذیر
  • ✅ طراحی توابع قابل استفاده در پروژه‌های واقعی

🧪 تمرین‌های پیشنهادی:

  1. تابعی بنویس که دو عدد بگیرد و حاصل‌ضرب آن‌ها را return کند.
  2. تابعی که یک عدد بگیرد و اگر زوج بود True برگرداند.
  3. تابعی که نام و نام خانوادگی گرفته و یک رشته کامل برگشتی بدهد.
  4. تابعی که لیستی از نمرات بگیرد و میانگین را return کند.
  5. تابعی بنویس که بیشترین مقدار در لیست را بازگرداند.
  6. تابعی که ساعت را گرفته و بر اساس آن سلام مناسب (صبح، ظهر، شب) بازگرداند.

🧾 جمع‌بندی:

کلمه کلیدی عملکرد
return مقدار خروجی تابع را مشخص می‌کند
خروجی می‌توان از آن در متغیر، شرط، حلقه، و... استفاده کرد
بدون return تابع مقدار None برمی‌گرداند
چند return می‌توان در تابع چند مسیر بازگشتی تعریف کرد

⚡️ توابع لامبدا (Lambda Functions) در پایتون

✅ تابع lambda چیست؟

تابع لامبدا (Lambda Function) یک نوع تابع کوچک، سریع و بدون نام (anonymous) است که برای کارهای ساده و کوتاه استفاده می‌شود.

✅ معمولاً وقتی فقط یک عمل ساده داریم (مثل ضرب، جمع، فیلتر)، بهتره از lambda استفاده کنیم.

🧱 ساختار تابع لامبدا:

lambda پارامترها: عبارت

مثل def نیست که کد بدنه چند خطی بنویسی. فقط یک خط خروجی دارد.

✅ مثال ساده:

add = lambda a, b: a + b
print(add(3, 5))  # خروجی: 8

در واقع این مثل تعریف معمولی زیره:

def add(a, b):
    return a + b

اما جمع‌وجورتر.

✅ لامبدا با پارامترهای مختلف:

حالت مثال توضیح
بدون پارامتر lambda: 3.14 همواره عدد 3.14 بازمی‌گرداند
یک پارامتر lambda x: x * 2 مقدار را دو برابر می‌کند
چند پارامتر lambda x, y: x + y جمع دو عدد

✅ استفاده از lambda همراه با توابع آماده:

1️⃣ با map(): اجرای یک عمل روی تمام اعضای لیست
nums = [1, 2, 3, 4]
doubled = list(map(lambda x: x * 2, nums))
print(doubled)
# خروجی: [2, 4, 6, 8]
2️⃣ با filter(): فیلتر کردن اعضا با شرط
nums = [1, 2, 3, 4, 5, 6]
evens = list(filter(lambda x: x % 2 == 0, nums))
print(evens)
# خروجی: [2, 4, 6]
3️⃣ با sorted() + کلید سفارشی:
students = [("Ali", 19), ("Sara", 17), ("Reza", 21)]
sorted_students = sorted(students, key=lambda x: x[1])
print(sorted_students)
# خروجی: [('Sara', 17), ('Ali', 19), ('Reza', 21)]

وضعیت مناسب استفاده از lambda
تابعی کوتاه با یک خط ✅ بله
نیاز به سرعت و خوانایی ✅ بله
استفاده موقتی در map/filter/sorted ✅ بله
چند خط کد یا منطق پیچیده ❌ نه (از def استفاده کن)

⚠️ محدودیت‌ها:

🧪 تمرین‌های پیشنهادی:

  1. تابع لامبدا بساز که عدد را سه برابر کند.
  2. با استفاده از map و lambda لیستی از اعداد را به توان ۲ برسان.
  3. با filter و lambda فقط اعداد منفی لیست را انتخاب کن.
  4. با sorted لیست نام‌ها را بر اساس طول آن‌ها مرتب کن.
  5. تابع لامبدایی بنویس که بررسی کند عدد زوج است یا نه.

ویژگی تابع lambda

مروری بر ماژول‌ها و پکیج‌ها در پایتون

ماژول‌ها و پکیج‌ها به شما امکان می‌دهند کد خود را سازماندهی کنید و از کدهای آماده دیگران استفاده کنید. این ویژگی یکی از قدرتمندترین جنبه‌های پایتون است.

ماژول چیست؟

ماژول یک فایل پایتون (.py) است که شامل تعاریف و دستورات پایتون است. این فایل می‌تواند توابع، کلاس‌ها و متغیرهایی را تعریف کند که می‌توانید در برنامه‌های دیگر استفاده کنید.

پکیج چیست؟

پکیج مجموعه‌ای از ماژول‌هاست که در یک پوشه قرار دارند. هر پوشه‌ای که شامل فایل __init__.py باشد، یک پکیج محسوب می‌شود.

مزایای استفاده از ماژول‌ها و پکیج‌ها:

  • سازماندهی بهتر کد
  • قابلیت استفاده مجدد از کد
  • جلوگیری از تداخل نام‌ها
  • استفاده از کتابخانه‌های آماده

انواع ماژول‌ها در پایتون:

نوع توضیح مثال
ماژول‌های استاندارد همراه با پایتون نصب می‌شوند math, random, os, sys
ماژول‌های شخص ثالث توسط جامعه پایتون توسعه یافته‌اند numpy, pandas, requests
ماژول‌های شخصی توسط خود شما نوشته می‌شوند my_module.py

نحوه استفاده:

import math
from datetime import datetime
from mypackage import mymodule

وارد کردن ماژول‌ها با import

برای استفاده از ماژول‌ها در پایتون، ابتدا باید آن‌ها را وارد (import) کنید. چند روش مختلف برای وارد کردن ماژول‌ها وجود دارد:

۱. وارد کردن کل ماژول

import math
print(math.sqrt(16))  # خروجی: 4.0

در این روش، باید نام ماژول را قبل از هر تابع یا متغیر ذکر کنید.

۲. وارد کردن با نام مستعار

import math as m
print(m.sqrt(16))  # خروجی: 4.0

این روش برای ماژول‌هایی با نام‌های طولانی مفید است.

۳. وارد کردن توابع یا کلاس‌های خاص

from math import sqrt
print(sqrt(16))  # خروجی: 4.0

در این روش، نیازی به ذکر نام ماژول قبل از تابع نیست.

۴. وارد کردن چند تابع یا کلاس

from math import sqrt, pow
print(sqrt(16))  # خروجی: 4.0
print(pow(2, 3))  # خروجی: 8.0

۵. وارد کردن همه محتویات (با احتیاط استفاده شود)

from math import *
print(sqrt(16))  # خروجی: 4.0
print(pi)        # خروجی: 3.141592653589793

⚠️ این روش می‌تواند باعث تداخل نام‌ها شود و توصیه نمی‌شود.

نکات مهم:

  • ماژول‌ها فقط یک بار بارگذاری می‌شوند، حتی اگر چندین بار import شوند.
  • ترتیب import‌ها مهم است. معمولاً ابتدا ماژول‌های استاندارد، سپس ماژول‌های شخص ثالث و در نهایت ماژول‌های محلی وارد می‌شوند.
  • import‌ها معمولاً در ابتدای فایل قرار می‌گیرند.

مثال کاربردی:

import random
import datetime
from math import sqrt

# تولید یک عدد تصادفی
random_number = random.randint(1, 100)

# نمایش تاریخ و زمان فعلی
current_time = datetime.datetime.now()

# محاسبه جذر عدد تصادفی
result = sqrt(random_number)

print(f"عدد تصادفی: {random_number}")
print(f"زمان فعلی: {current_time}")
print(f"جذر عدد: {result}")

توابع در پایتون

🔹 1. تعریف تابع با def

تابع‌ها بلوک‌هایی از کدن که می‌تونن چند بار اجرا بشن و ساختار مشخصی دارن.

ساختار:

def name():
    # بدنه‌ی تابع

📌 برای انجام کاری مشخص و جلوگیری از تکرار کد استفاده می‌شن.

مثال:

def say_hello():
    print("سلام!")

🔹 2. پارامترها و آرگومان‌ها

پارامترها: متغیرهایی که موقع تعریف تابع مشخص می‌شن.

آرگومان‌ها: مقادیری که هنگام فراخوانی تابع ارسال می‌شن.

مثال:

def greet(name):
    print("سلام", name)

greet("علی")  # name = "علی"

📌 می‌تونن:

  • مقدار پیش‌فرض داشته باشن
  • تعداد نامشخص داشته باشن: *args, **kwargs

🔹 3. مقادیر بازگشتی با return

تابع می‌تونه نتیجه‌ای تولید کنه و برگردونه با استفاده از return.

مثال:

def add(a, b):
    return a + b

x = add(3, 5)  # x = 8

📌 با return تابع خروجی داره که می‌تونی ذخیره، چاپ یا استفاده کنی.

📌 بعد از اجرای return، تابع متوقف می‌شه.

🔹 4. توابع لامبدا lambda

توابع کوچک و بدون نام برای محاسبات ساده – فقط یک خط خروجی دارن.

ساختار:

lambda پارامترها: عبارت

مثال:

add = lambda x, y: x + y
print(add(2, 3))  # خروجی: 5

📌 خیلی مفیدن در ترکیب با: map, filter, sorted

📌 برای عملیات سریع و سبک استفاده می‌شن.

🧠 یادآوری سریع:

بخش کلید اصلی
def تعریف تابع
پارامتر / آرگومان داده ورودی برای تابع
return خروجی دادن از تابع
lambda تابع کوچک و سریع بدون اسم

⚠️ مدیریت خطاها و استثناها

🔍 چرا باید خطاها رو مدیریت کنیم؟

در پایتون، اگر خطایی (مثلاً تقسیم بر صفر یا باز کردن فایل ناموجود) رخ بده، برنامه به صورت پیش‌فرض کرش می‌کنه.

✅ با مدیریت خطاها:

  • برنامه به‌جای کرش، پیام مناسب می‌ده
  • کنترل در دست ما باقی می‌مونه
  • تجربه کاربر بهتر می‌شه

🏗️ ساختار try / except

try:
    # کدی که احتمال خطا در آن هست
except:
    # کدی که در صورت بروز خطا اجرا می‌شود

📝 مثال : تقسیم بر صفر

try:
    result = 10 / 0
except:
    print("خطا: تقسیم بر صفر امکان‌پذیر نیست.")

💡 اگر خطا رخ بده، برنامه وارد except می‌شه و پیام مناسب چاپ می‌کنه، نه اینکه متوقف بشه.

🎯 مشخص‌کردن نوع خطا (Exception Type)

برای مدیریت دقیق‌تر می‌تونیم نوع خاصی از خطا رو بگیریم:

try:
    num = int(input("عدد وارد کن: "))
except ValueError:
    print("خطا: لطفاً فقط عدد صحیح وارد کن.")

🔧 ساختار کامل try / except / else / finally

try:
    # کد اصلی
except ExceptionType:
    # مدیریت خطا
else:
    # اگر خطا رخ نده، این بخش اجرا میشه
finally:
    # همیشه اجرا میشه (چه خطا باشه یا نه)

💻 مثال : استفاده از else و finally

try:
    x = int(input("عدد اول: "))
    y = int(input("عدد دوم: "))
    result = x / y
except ZeroDivisionError:
    print("خطا: تقسیم بر صفر مجاز نیست.")
except ValueError:
    print("خطا: ورودی عدد نیست.")
else:
    print("نتیجه:", result)
finally:
    print("پایان عملیات.")

📋 لیستی از خطاهای رایج در پایتون:

خطا توضیح
ZeroDivisionError تقسیم بر صفر
ValueError تبدیل نوع نامعتبر
TypeError استفاده اشتباه از نوع داده
IndexError دسترسی به اندیس ناموجود در لیست
KeyError کلید ناموجود در دیکشنری
FileNotFoundError فایل پیدا نشد
NameError استفاده از متغیر تعریف‌نشده

🚀 raise: ایجاد دستی خطا

age = -5
if age < 0:
    raise ValueError("سن نمی‌تواند منفی باشد.")

💡 گاهی وقتا خودت می‌خوای شرایط خاصی رو کنترل کنی.

🛠️ تعریف استثنا (Exception) سفارشی

class MyError(Exception):
    pass

def check(x):
    if x > 100:
        raise MyError("عدد خیلی بزرگه!")

try:
    check(150)
except MyError as e:
    print("خطای سفارشی:", e)

📚 تمرین‌های پیشنهادی:

  1. برنامه‌ای بنویس که دو عدد از کاربر بگیره و حاصل تقسیم رو چاپ کنه؛ خطاهای ZeroDivisionError و ValueError رو مدیریت کن.
  2. برنامه‌ای که یک فایل متنی رو باز کنه؛ اگر فایل وجود نداشت FileNotFoundError رو مدیریت کنه.
  3. لیستی تعریف کن و از کاربر اندیسی بگیر؛ اگر اندیس نامعتبر بود، IndexError چاپ کن.
  4. دیکشنری داشته باش و کلیدی که وجود نداره رو فراخوانی کن؛ KeyError رو مدیریت کن.
  5. تابعی بنویس که اگر عدد ورودی بیشتر از ۱۰۰ باشه، خطای سفارشی TooBigError بندازه.

📝 جمع‌بندی سریع:

دستور توضیح
try بخشی که ممکنه خطا داشته باشه
except واکنش در صورت خطا
else اگر هیچ خطایی نباشه، اجرا میشه
finally همیشه اجرا میشه، حتی اگر خطا رخ بده
raise ایجاد دستی خطا
Exception types مثل ZeroDivisionError, ValueError, FileNotFoundError, ...

🧬 وراثت (Inheritance) در پایتون

🔹 وراثت یعنی چی؟

وراثت (Inheritance) در OOP یعنی یک کلاس می‌تونه ویژگی‌ها و متدهای یک کلاس دیگر را به ارث ببرد.

  • به کلاسی که ویژگی‌ها را به ارث می‌دهد، می‌گوییم: کلاس والد (Parent) یا پایه (Base Class)
  • به کلاسی که ارث‌بری می‌کند، می‌گوییم: کلاس فرزند (Child) یا مشتق (Derived Class)

🔹 چرا از وراثت استفاده می‌کنیم؟

  • جلوگیری از تکرار کد
  • قابلیت گسترش و سازمان‌دهی بهتر
  • ساده‌تر شدن تغییرات در پروژه‌های بزرگ
  • ساخت ساختار سلسله‌مراتبی (Hierarchical)

🔹 نحوه تعریف وراثت در پایتون:

class Parent:
    # ویژگی‌ها و متدهای کلاس پایه

class Child(Parent):
    # کلاس فرزند از Parent ارث‌بری می‌کند

🔹 مثال: وراثت ساده

class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        print(f"{self.name} صدا می‌دهد.")

# کلاس فرزند
class Dog(Animal):
    pass

dog1 = Dog("سگ")
dog1.speak()   # خروجی: سگ صدا می‌دهد.

نکته: چون Dog از Animal ارث‌بری کرده، متد speak() و ویژگی name را دارد، حتی اگر در کلاس Dog چیزی ننوشته باشیم!

🔹 مثال: افزودن یا بازنویسی در کلاس فرزند

class Cat(Animal):
    def speak(self):
        print(f"{self.name} میو می‌کند.")

cat1 = Cat("گربه")
cat1.speak()  # خروجی: گربه میو می‌کند.

اینجا متد speak() در کلاس Cat بازنویسی (Override) شده.

🔹 استفاده از super() برای ارث‌بری صحیح‌تر

اگر بخواهی در کلاس فرزند از __init__ کلاس والد استفاده کنی:

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

class Student(Person):
    def __init__(self, name, age, grade):
        super().__init__(name, age)  # ارجاع به __init__ کلاس والد
        self.grade = grade

حالا Student هم name و age داره، هم grade.

🔹 تفاوت Override و Extend

نوع معنی مثال
Override بازنویسی متد والد در فرزند متد speak() در کلاس Cat
Extend اضافه‌کردن متد یا ویژگی جدید در کلاس فرزند ویژگی grade در کلاس Student

🔹 تمرین‌های پیشنهادی:

  1. کلاس Vehicle بساز با متد move()، سپس کلاس Car بساز که از آن ارث ببرد.
  2. کلاس Person با ویژگی‌های name و age بساز، سپس کلاس Employee تعریف کن که salary هم داشته باشد.
  3. کلاس Animal بساز و از آن کلاس‌های Bird و Fish بساز که fly() و swim() مخصوص خودشون رو داشته باشن.
  4. کلاس Shape بساز و از آن Rectangle و Circle ارث‌بری کن که هر کدام متد area() مخصوص خودشون داشته باشن.
  5. در کلاس فرزند از super().__init__() استفاده کن تا کد کلاس والد را دوباره استفاده کنی.

🔹 جمع‌بندی سریع:

مفهوم توضیح
وراثت (Inheritance) ارث‌بری ویژگی‌ها و رفتارها از یک کلاس دیگر
Parent Class کلاس پایه (مثلاً Animal)
Child Class کلاس فرزند (مثلاً Dog)
super() فراخوانی اجزای کلاس والد داخل کلاس فرزند
Override بازنویسی متد والد در کلاس فرزند
استفاده ساخت ساختار تمیز، قابل توسعه و بدون تکرار کدها

🔒 کپسوله‌سازی (Encapsulation) در پایتون

🔹 کپسوله‌سازی یعنی چی؟

Encapsulation = مخفی‌سازی داده‌ها + کنترل دسترسی به آن‌ها

یعنی:

  • داده‌های (ویژگی‌های) شیء را مستقیم در دسترس همه قرار ندهیم
  • و دسترسی به آن‌ها را از طریق متدها (توابع داخل کلاس) مدیریت کنیم

🔹 چرا از کپسوله‌سازی استفاده می‌کنیم؟

  • امنیت بیشتر
  • جلوگیری از تغییر ناخواسته داده‌ها
  • کنترل دقیق روی مقداردهی و دسترسی
  • سازمان‌دهی بهتر کد

🔹 سطوح دسترسی در پایتون:

سطح پیشوند توضیح
عمومی (Public) بدون پیشوند همه جا قابل دسترسی
محافظت‌شده (Protected) _متغیر به‌طور قراردادی فقط داخل کلاس و زیرکلاس‌ها استفاده می‌شه
خصوصی (Private) __متغیر فقط داخل خود کلاس قابل استفاده (با محدودیت بیشتر)

🔹 مثال: ویژگی عمومی (Public)

class Person:
    def __init__(self, name):
        self.name = name

p = Person("Ali")
print(p.name)  # خروجی: Ali

این ویژگی بدون محدودیت از بیرون کلاس قابل دسترسی است.

🔹 مثال: ویژگی خصوصی با __ (Private)

class Person:
    def __init__(self, name):
        self.__name = name  # خصوصی

p = Person("Ali")
print(p.__name)  # ❌ ارور: قابل دسترسی نیست

اما با متدهای داخلی (getter/setter) می‌تونیم دسترسی کنترل‌شده بدیم 👇

🔹 متدهای getter و setter:

  • getter: برای گرفتن مقدار
  • setter: برای تغییر مقدار با بررسی و امنیت

🔹 مثال: استفاده از getter و setter

class Person:
    def __init__(self, name):
        self.__name = name

    def get_name(self):  # getter
        return self.__name

    def set_name(self, new_name):  # setter
        if len(new_name) >= 2:
            self.__name = new_name
        else:
            print("نام باید حداقل 2 حرف باشد.")

p = Person("Ali")
print(p.get_name())  # ✅ دسترسی به name

p.set_name("A")      # ❌ خطا: نام خیلی کوتاه است
p.set_name("Sara")   # ✅ تغییر موفق
print(p.get_name())  # خروجی: Sara

🔹 نکته مهم درباره پایتون:

پایتون محدودیت سطح دسترسی واقعی مثل Java یا C++ نداره، اما با قراردادهای نام‌گذاری (مثل __var) به برنامه‌نویس می‌فهمونه که فلان داده خصوصی هست.

🔹 تمرین‌های پیشنهادی:

  1. کلاسی به نام BankAccount بساز که موجودی را به‌صورت خصوصی نگه دارد و متدهایی برای واریز، برداشت و نمایش موجودی داشته باشد.
  2. کلاس User بنویس که ایمیل را خصوصی نگه دارد و فقط از طریق متد get_email() قابل مشاهده باشد.
  3. کلاس Car بساز که سرعت ماشین را با متد set_speed() مقداردهی کند و فقط در بازه‌ی 0 تا 200 مجاز باشد.
  4. کلاس Product بساز که نام و قیمت را خصوصی نگه دارد و بررسی کند قیمت منفی نباشد.
  5. کلاس Student طراحی کن که نمره را فقط از طریق set_grade() تعیین کند و از دستکاری مستقیم جلوگیری شود.

🎭 پلی‌مورفیسم (Polymorphism) در پایتون

🔹 پلی‌مورفیسم یعنی چی؟

Polymorphism یعنی «چندشکلی» → توانایی استفاده از یک رابط یکسان برای اشیاء مختلف با رفتار متفاوت.

یعنی متد یا تابعی با نام یکسان، اما رفتار متفاوت در کلاس‌های مختلف داشته باشه.

🔹 مثال واقعی:

فرض کن متد speak() رو داریم:

  • برای Dog می‌گه "Woof"
  • برای Cat می‌گه "Meow"
  • برای Cow می‌گه "Moo"

هر سه شیء متد speak() دارن ولی خروجی متفاوت! این یعنی چندشکلی.

🔹 انواع پیاده‌سازی پلی‌مورفیسم در پایتون

  1. پلی‌مورفیسم در متدهای کلاس‌های مختلف
  2. پلی‌مورفیسم با وراثت و Override متدها
  3. پلی‌مورفیسم در توابع و حلقه‌ها (رفتار یکسان روی اشیاء مختلف)

🔹 مثال: متدهای با نام یکسان در کلاس‌های مختلف

class Dog:
    def speak(self):
        print("Woof")

class Cat:
    def speak(self):
        print("Meow")

# فراخوانی یکسان، رفتار متفاوت
d = Dog()
c = Cat()

d.speak()  # Woof
c.speak()  # Meow

🔹 مثال: استفاده از پلی‌مورفیسم در حلقه

animals = [Dog(), Cat()]

for animal in animals:
    animal.speak()

بدون اینکه بدونیم دقیقاً چه نوع حیوانی هست، می‌دونیم که هرکدام متدی به نام speak() دارند.

🔹 مثال: وراثت + Override

class Animal:
    def speak(self):
        print("صدا...")

class Dog(Animal):
    def speak(self):
        print("سگ: هاپ هاپ")

class Cat(Animal):
    def speak(self):
        print("گربه: میو")

def make_sound(animal):
    animal.speak()

make_sound(Dog())  # سگ: هاپ هاپ
make_sound(Cat())  # گربه: میو

تابع make_sound() با هر شیء کار می‌کنه چون متد speak() رو همه دارن = این یعنی پلی‌مورفیسم!

🔹 مثال: پلی‌مورفیسم در توابع معمولی

def add(x, y):
    return x + y

print(add(5, 10))        # 15
print(add("Hello ", "World"))  # Hello World
print(add([1, 2], [3]))  # [1, 2, 3]

تابع add() برای عدد، رشته و لیست کار می‌کنه → اینم شکلی از پلی‌مورفیسم در سطح تابع.

🔹 چرا پلی‌مورفیسم مهمه؟

مزیت توضیح
🔄 بازنویسی راحت متدها کلاس‌های فرزند می‌تونن متد والد رو تغییر بدن
🚀 سادگی در توسعه توابع می‌تونن با اشیاء مختلف بدون تغییر کار کنن
🔧 افزایش انعطاف وابستگی به نوع دقیق شیء کمتر میشه
📐 اصل Open/Closed کد برای تغییر بسته است، برای توسعه باز است

🔹 تمرین‌های پیشنهادی:

  1. کلاس‌های Bird, Fish, Dog بنویس که متدی به نام move() داشته باشن با رفتار متفاوت.
  2. کلاس والد Shape با متد area() بنویس و کلاس‌های Circle و Square از اون ارث ببرن و متد area() رو بازنویسی کنن.
  3. تابعی بنویس که یک شیء بگیره و متد show_info() رو صدا بزنه؛ اشیاء مختلفی مثل User, Product, Employee با همین متد بساز.
  4. یک لیست شامل اشیاء مختلف بساز (مثلاً حیوانات)، و با یک حلقه for متد مشابهی روی همه اجرا کن.
  5. کلاس Transport بساز و از اون Car, Bike, Bus ارث‌بری کن که متد speed() داشته باشن؛ خروجی‌ها متفاوت باشن.

Override بازنویسی متد کلاس والد در کلاس فرزند
کاربرد ساده‌سازی توابع، حلقه‌ها، و ساختار OOP
